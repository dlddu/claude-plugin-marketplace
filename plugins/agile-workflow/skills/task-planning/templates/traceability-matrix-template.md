# 추적성 매트릭스 (Traceability Matrix)

이 문서는 요구사항부터 구현까지의 모든 산출물을 양방향으로 추적합니다.

## 개요

양방향 추적성 매트릭스(Bidirectional Traceability Matrix)는 다음을 보장합니다:
- **순방향 추적**: 요구사항 → 명세 → 사용자 스토리 → 작업 → 구현
- **역방향 추적**: 구현 → 작업 → 사용자 스토리 → 명세 → 요구사항

## 요구사항 → 구현 추적

| REQ ID | SPEC ID | US ID | AT ID | TASK ID | TC ID | 구현 상태 | CI 상태 | 완료일 |
|--------|---------|-------|-------|---------|-------|---------|--------|-------|
| REQ-001 | SPEC-001 | US-001 | AT-001.1 | TASK-001 | TC-001 | ⬜ | - | - |
| REQ-001 | SPEC-001 | US-001 | AT-001.2 | TASK-001 | TC-002 | ⬜ | - | - |

## ID 규칙

### 식별자 형식
- **REQ**: 원본 요구사항 (REQ-001, REQ-002, ...)
- **SPEC**: 기능 명세서 (SPEC-001, SPEC-002, ...)
- **US**: 사용자 스토리 (US-001, US-002, ...)
- **AT**: 인수 테스트 (AT-001.1, AT-001.2, ...)
- **TASK**: 작업 계획서 (TASK-001, TASK-002, ...)
- **TC**: 테스트 케이스 (TC-001, TC-002, ...)

### 연결 규칙
```
REQ-001 ─┬─ SPEC-001 ─┬─ US-001 ─┬─ AT-001.1 ─── TASK-001 ─── TC-001
         │            │          ├─ AT-001.2 ─── TASK-001 ─── TC-002
         │            │          └─ AT-001.3 ─── TASK-002 ─── TC-003
         │            │
         │            └─ US-002 ─┬─ AT-002.1 ─── TASK-003 ─── TC-004
         │                       └─ AT-002.2 ─── TASK-003 ─── TC-005
         │
         └─ SPEC-002 ─── ...
```

## 상태 표시

### 구현 상태
| 기호 | 의미 | 설명 |
|-----|-----|------|
| ⬜ | 미착수 | 아직 작업이 시작되지 않음 |
| 🔄 | 진행중 | 현재 작업 중 |
| ✅ | 완료 | 구현 및 테스트 완료 |
| ❌ | 실패 | 테스트 실패 또는 블록됨 |
| ⏸️ | 보류 | 외부 의존성으로 대기 중 |

### CI 상태
| 기호 | 의미 |
|-----|------|
| ✅ | 모든 CI 체크 통과 |
| ❌ | CI 체크 실패 |
| 🔄 | CI 실행 중 |
| - | 아직 CI 실행 안함 |

## 문서 간 연결

### 명세서 (SPEC)
- 위치: `docs/specs/SPEC-{번호}-{기능명}.md`
- 연결: REQ → SPEC → US

### 사용자 스토리 (US)
- 위치: `docs/user-stories/US-{번호}-{기능명}.md`
- 연결: SPEC → US → AT

### 인수 테스트 (AT)
- 위치: 사용자 스토리 문서 내 "인수 테스트" 섹션
- 연결: US → AT → TASK → TC

### 작업 계획서 (TASK)
- 위치: `docs/tasks/TASK-{번호}-{작업명}.md`
- 연결: AT → TASK → TC

### 테스트 케이스 (TC)
- 위치: 소스 코드 테스트 파일
- 연결: TASK → TC

## 사용 방법

### 새 요구사항 추가 시
1. REQ ID 부여
2. SPEC 문서 작성 및 SPEC ID 부여
3. US 문서 작성 및 US ID 부여
4. AT 작성 및 AT ID 부여
5. 이 매트릭스에 새 행 추가

### 구현 진행 시
1. TASK 작성 및 TASK ID 부여
2. 구현 및 TC 작성
3. 구현 상태 업데이트 (⬜ → 🔄)
4. CI 통과 시 상태 업데이트 (🔄 → ✅)
5. 완료일 기록

### 테스트 실패 시
1. 구현 상태를 ❌로 변경
2. 원인 분석 후 수정
3. CI 재실행
4. 통과 시 ✅로 변경
